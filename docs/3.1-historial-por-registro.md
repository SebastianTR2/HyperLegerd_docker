# 3.1 Historial por registro

Este documento detalla la implementación del historial de modificaciones para los activos (Assets) y los tokens (ERC-20) en la red Hyperledger Fabric del proyecto.

## 1. Recuperación de historial de cambios para Asset
Se ha añadido la capacidad de visualizar todas las modificaciones a lo largo del tiempo de un cliente registrado.
- **Chaincode (`cliente_cc`)**: Se agregó la función `GetAssetHistory`, la cual utiliza `GetHistoryForKey` de la API nativa de Fabric. Esta función recupera cada uno de los cambios, incluyendo la creación inicial, cualquier actualización y si el registro fue borrado.
- **Middleware**: Se expuso este historial mediante un nuevo endpoint GET `/clientes/historial/{clienteId}`.

## 2. Recuperación de historial relevante para Token
Se ha extendido el contrato de token básico para permitir consultar el histórico de operaciones.
- **Chaincode (`token_erc20`)**: Se creó la función `GetHistory` que lee el ciclo de vida del balance de una cuenta en específico a lo largo del tiempo.
- **Middleware**: Se actualizó el endpoint GET `/tokens/historial/{clienteId}?codigoToken=TOK` que previamente devolvía el error HTTP 501. Ahora contacta al chaincode para devolver los eventos históricos del balance.

## 3. Ordenamiento por secuencia temporal
Por defecto, Hyperledger Fabric devuelve el historial del evento más antiguo al más nuevo.
Para facilitar la presentación desde el frontend:
- Ambas funciones en los chaincodes (`GetAssetHistory` y `GetHistory`) implementan un algoritmo que **invierte** (reverse) el arreglo antes de enviarlo.
- De esta manera, las interfaces de usuario obtendrán siempre los datos ordenados en orden cronológico descendente (es decir, **los eventos más recientes primero**).

## 4. Formateo para consumo desde la interfaz
Para que los desarrolladores de frontend puedan pintar de manera directa los datos en la pantalla (como en tablas), el API Middleware estructura las respuestas crudas de la blockchain a un formato consistente:
- Se estandarizó la respuesta añadiendo el `TxId` (identificador de la transacción), `Timestamp` (cuándo ocurrió el cambio), y el `record` (datos asociados).
- En el caso de los tokens, se adaptaron las propiedades a un formato estandarizado que devuelve un arreglo en la propiedad `operaciones`.

## Uso Práctico desde la Interfaz / API

### Obtener el Historial de un Cliente (Asset)
```bash
curl -X GET "http://localhost:3000/clientes/historial/CLI001" -H "X-API-Key: sec-admin"
```

### Obtener el Historial de Balance de un Token
```bash
curl -X GET "http://localhost:3000/tokens/historial/CLI001?codigoToken=TOK" -H "X-API-Key: sec-admin"
```
