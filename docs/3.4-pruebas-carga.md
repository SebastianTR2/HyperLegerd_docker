# 3.4 Pruebas de carga (entregable Sprint 3)

Cierra el último entregable del Sprint 3: **pruebas de carga ejecutadas con reporte de estabilidad**.

## 1. Alcance

Carga sintética sobre el `api-middleware` validando:

1. **Estabilidad multi-tenant**: dos empresas (clientes + agricultura) operando en paralelo no se interfieren.
2. **Latencia** dentro de umbrales razonables para una red Fabric test-network local.
3. **Tasa de error** menor al 5 %.

## 2. Herramienta

- [**k6**](https://k6.io/) (open source, soporta escenarios concurrentes, métricas y umbrales).
- Alternativas equivalentes: `vegeta`, `wrk`, `JMeter`. El script puede portarse fácilmente.

## 3. Script

Ubicación: `pruebas-carga/k6-tenants.js`  
Documentación: `pruebas-carga/README.md`

## 4. Escenarios

| Escenario | Tenant | Operación | VUs | Duración |
|-----------|--------|-----------|-----|----------|
| `lectura_clientes` | `clientes` (legacy) | `GET /clientes` | 5 constantes | 30 s |
| `escritura_agricultura` | `agricultura` | `POST /datos` | rampa 0→5 | 40 s |
| `lectura_agricultura` | `agricultura` | `GET /datos` | 4 constantes | 40 s |

Total: 3 escenarios concurrentes, hasta 14 usuarios virtuales en pico.

## 5. Umbrales (thresholds)

| Métrica | Umbral |
|---------|--------|
| `http_req_failed` global | `< 5 %` |
| p95 lecturas (`clientes` y `agricultura`) | `< 1500 ms` |
| p95 escrituras (`agricultura`) | `< 3000 ms` |

Las escrituras admiten más latencia porque incluyen el ciclo completo **Execute-Order-Validate-Commit** de Fabric.

## 6. Ejecución

```bash
# 1. Asegurar middleware + Fabric + chaincodes desplegados
# 2. Ejecutar k6
k6 run pruebas-carga/k6-tenants.js

# Exportar resumen para el informe
k6 run --summary-export=reporte-carga.json pruebas-carga/k6-tenants.js
```

## 8. Mitigaciones si falla algún umbral

| Síntoma | Acción |
|---------|--------|
| p95 escrituras > 3 s | Revisar política de endoso (¿necesita 2 firmas? bajar a `OR(Org1)` para test). |
| Errores 502 / 503 | Peer caído o saturado: `docker stats`, reiniciar. |
| Errores MVCC | Reducir concurrencia o serializar updates sobre la misma clave. |
| Errores 401/403 | Verificar API keys en `tenants.yaml`. |
